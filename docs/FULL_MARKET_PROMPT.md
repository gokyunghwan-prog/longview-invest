# Longview 국내 전체시장 수집 프롬프트

KOSPI·KOSDAQ 전체 상장사를 Open DART 공식 API로 매일 갱신하는 파이프라인을 구현하라.

## 수집 범위

- `corpCode.xml`에서 종목코드가 있는 법인을 시작점으로 삼는다.
- `company.json`의 `corp_cls=Y`는 KOSPI, `corp_cls=K`는 KOSDAQ으로 분류한다.
- KONEX와 비상장 법인은 공개 목록에서 제외한다.
- 목록을 임의 종목 샘플로 제한하지 않는다.
- 신규사나 재무가 부족한 회사도 목록에 남기고 `insufficient_data`로 표시한다.

## 증분 수집

- 전체 법인코드를 한 번 내려받아 체크포인트에 저장하고 `modify_date`가 바뀐 법인만
  기업개황을 다시 확인한다.
- 최근 공시는 `list.json`을 짧은 날짜 구간으로 나눠 증분 수집한다.
- 재무는 고유번호를 최대 100개씩 묶어 다중회사 주요계정·재무지표 API로 받는다.
- 최근 제출 가능한 보고서를 우선하되 장기 점수는 기간이 맞는 연차 재무를 우선한다.
- 연결 기준을 우선하고 연결 자료가 없을 때만 별도 기준을 사용한다.
- 한 회사의 오류가 전체 배치를 중단시키지 않도록 실패를 기록하고 다음 회사로 진행한다.

## 표준화와 계보

- 회사 ID는 국내 법인 고유번호를 기반으로 안정적으로 만든다.
- 6자리 종목코드의 앞자리 0을 보존한다.
- 매출, 영업이익, 순이익, 자산, 부채, 자본과 3개년 추이를 같은 단위로 정규화한다.
- 정정·철회 공시를 표시하고 최신 접수번호와 원문 URL을 남긴다.
- 공식 자료에 없는 값은 추정하지 않는다.

## 출력과 검증

- 임시 파일에 완성한 뒤 전체 JSON 파싱, 중복 ID, 국내 회사만 포함됐는지, 최소 기업 수,
  시장별 커버리지와 비정상 급감을 검증한다.
- 검증을 통과한 경우에만 `data/companies.json`을 원자적으로 교체한다.
- 실패하면 마지막 정상 파일과 DART 체크포인트를 보존한다.
- 진단에는 비밀키, 전체 요청 URL, 원문 응답을 남기지 않는다.
- 원격 또는 과거 파일이 섞여 있어도 병합 전에 국내 회사만 정규화한다.

## 자동화

- GitHub Actions에서 코드 검사 → DART 수집 → 국내 시세 결합 → 스냅샷 검증 → 데이터
  파일 커밋 순으로 실행한다.
- DART 인증키는 Repository Secret으로만 전달한다.
- 같은 작업이 겹치지 않도록 concurrency와 파일 잠금을 사용한다.
- 중간에 끊겨도 다음 실행에서 체크포인트를 이어 사용한다.

완료 기준은 전체시장 포함, 추적 가능한 공시 계보, 실패 시 정상 데이터 보존,
`npm run check` 및 스냅샷 validator 통과다.
