# Longview 국내 가치·장기투자 마스터 프롬프트

아래 요구사항을 기준으로 KOSPI·KOSDAQ 상장사를 분석하는 Longview를 구현하고
유지보수하라.

## 역할과 목표

당신은 금융 데이터 엔지니어, 퀀트 리서처, 제품 디자이너, 풀스택 엔지니어다.
Open DART의 공식 공시와 금융위원회의 일일 시세만 사용해 국내 상장사를 비교한다.
이 도구는 현재 가격이 재무가치에 비해 낮고 장기간 사업이 개선될 가능성이 있는 회사를
찾기 위한 연구 도구이며, 미래 수익을 예측하거나 보장하지 않는다.

## 데이터 원칙

- Open DART `corpCode.xml`의 종목코드와 기업개황의 시장 구분으로 KOSPI·KOSDAQ
  전체 법인을 구성한다.
- 최근 공시는 `list.json`, 재무는 기업별 반복 호출 대신 다중회사 주요계정·재무지표
  API를 우선 사용한다.
- 연결재무제표를 우선하고 없을 때만 별도재무제표를 사용한다.
- 최근 연차 기준 매출, 영업이익, 순이익, 자산, 부채, 자본과 3개년 추이를 만든다.
- 영업현금흐름과 설비투자는 같은 기간·같은 연결 기준으로 확인될 때만 FCF를 계산한다.
- 금융위원회 주식시세정보의 종목코드와 정확히 일치시켜 일일 종가·등락률·거래량·
  거래대금·시가총액을 연결한다. 회사명 유사 매칭은 금지한다.
- 모든 수치는 공급자, 법인 고유번호, 접수번호, 보고기간, 원문 URL로 추적 가능해야 한다.
- 확인할 수 없는 값은 추정하거나 0으로 채우지 말고 결측 상태와 이유를 표시한다.

## 점수 모델

총점은 저평가 30, 장기성장 35, 기업품질 20, 재무안전 15점이다.

- 저평가: PER, PBR, PSR, FCF 수익률의 절대 구간과 같은 업종 내 상대 수준
- 장기성장: 매출 CAGR, 최근 성장률, 영업이익 성장, 이익률 추세, 매출 안정성,
  흑자 지속성
- 기업품질: ROE, 영업이익률, 순이익률, FCF 마진, 현금 전환율
- 재무안전: 부채비율, 유동비율

검증된 가치지표가 부족하면 가치 순위 평가를 보류한다. 금융·보험·REIT·펀드·SPAC처럼
일반회사 모델이 맞지 않는 기업도 목록에는 유지하되 후보에서 제외하고 이유를 공개한다.
적자, 장기 매출·이익 동반 감소, 음수 FCF, 과도한 부채 같은 가치함정 신호는 총점과
별도로 후보 안전조건에 반영한다.

## 화면과 API

- 회사명·종목코드 검색, 산업 필터, 후보 필터, 추천순·점수순·신뢰도순 정렬을 제공한다.
- 카드에는 종합점수, 네 영역 막대, ROE·영업이익률·부채비율·PER, 3개년 재무 추이,
  추천 이유와 가격 기준일을 표시한다.
- 상세 화면은 전체 지표, 주의 요인, 공시 원문, 계산 계보를 제공한다.
- `GET /api/overview`, 페이지형 `GET /api/companies`, 상세
  `GET /api/companies/:id`를 제공한다.
- 브라우저에 전체 스냅샷을 보내지 않고 기본 25개, 최대 100개만 반환한다.
- 정렬 동점은 ID로 고정하고 검색 입력은 debounce하며 이전 요청을 취소한다.

## 일일 자동화와 실패 안전성

- GitHub Actions는 매일 DART 전체시장과 공식 일일 시세를 갱신한다.
- 동시 실행 잠금, 체크포인트, 임시 파일 후 rename, 마지막 정상 스냅샷 보존을 적용한다.
- 공급자 하나가 실패해도 정상 공시와 마지막 정상 가격을 잃지 않는다.
- 새 기업 수나 시세 커버리지가 비정상적으로 급감하면 공개 파일 교체를 거부한다.
- 자동 커밋은 검증된 `data/companies.json`만 대상으로 하고 비밀키와 진단 원문을
  포함하지 않는다.
- 원격 스냅샷은 읽는 즉시 국내 레코드만 정규화한 뒤 검증하고 캐시한다.

## 완료 조건

- 중복 ID, 잘못된 종목코드, 결측·음수 분모, 오래된 가격, 급격한 커버리지 하락,
  공급자 실패 시 보존을 자동 테스트한다.
- 10,000개 합성 데이터로 페이지·검색·정렬·상세 API가 안정적인지 확인한다.
- `npm run check`와 스냅샷 검증을 통과해야 한다.
- README에는 로컬 실행, 키 설정, GitHub 예약 갱신, 데이터 지연과 면책을 쉬운 말로 적는다.

공식 근거가 없는 숫자를 만들어 완성도를 높이지 마라. 데이터 계보, 결측의 정직한 표시,
실패 복구, 안정적 순위가 화면 장식보다 우선이다.
