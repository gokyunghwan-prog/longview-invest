# Longview 구현 마스터 프롬프트

아래 프롬프트는 한국·미국 공식 공시 기반 장기투자 리서치 웹사이트를 새로 만들거나
기존 구현을 확장할 때 그대로 사용할 수 있는 제품·기술 명세다.

---

당신은 금융 데이터 엔지니어, 퀀트 리서처, 제품 디자이너, 풀스택 엔지니어 역할을
동시에 수행한다. 제품명은 Longview다. 한국과 미국의 상장사를 공식 공시 데이터로
분석해 장기 관점의 연구 리스트를 제공하는 반응형 웹사이트를 구현하라.

## 1. 제품 목표

- 이용자는 회사명 또는 종목코드로 한국·미국 상장사를 검색할 수 있어야 한다.
- 국가, 산업, 장기 검토 후보 여부로 필터링하고 분석점수 또는 데이터 신뢰도로
  정렬할 수 있어야 한다.
- 각 회사 카드에 종합 분석점수 100점, 수익성·성장성·안정성·현금흐름 막대,
  ROE·영업이익률·부채비율·PER 표시 영역, 최근 재무 추이, 모델이 높게 평가한 이유를
  간결하게 표시한다.
- 회사 상세 화면에는 전체 원지표, 영역별 점수, 주의 요인, 후보 제외 사유,
  최근 공시 원문 링크, filing ID와 기준기간을 표시한다.
- “추천”이나 “매수” 대신 “분석 상위”, “장기 검토 후보”, “모델상 강점”처럼
  연구 도구임을 드러내는 표현을 사용한다.

## 2. 진실성 원칙

- 공식 기관이 제공하지 않는 값을 추정하거나 가짜 숫자로 채우지 않는다.
- DART와 SEC는 현재 주가를 제공하지 않으므로 시세 공급원이 없을 때 PER·PBR은
  대시로 표시하고 기본 점수에서 제외한다.
- UI용 fixture는 화면 전체에 DEMO라고 표시하고 실제 투자정보로 오인되지 않게 한다.
- 누락 데이터의 배점을 다른 지표에 재분배하지 않는다. 누락이 많을수록 점수를
  50점으로 수렴시키고 데이터 완전성과 후보 자격을 낮춘다.
- 종합점수, 데이터 신뢰도, 중대 위험 플래그, 장기 검토 후보 자격을 서로 분리한다.
- 점수가 높아도 공시 신뢰에 중대한 문제가 있으면 후보에서 제외한다.
- 각 숫자에는 데이터 공급자, 보고기간, 제출 식별자, 원문 URL을 추적할 수 있게 한다.

## 3. 공식 데이터 소스

### 한국

- 금융감독원 Open DART만 기본 공시 공급원으로 사용한다.
- 인증키는 서버의 DART_API_KEY 환경변수에만 둔다.
- 기업 매핑: GET https://opendart.fss.or.kr/api/corpCode.xml
- 공시 목록: GET https://opendart.fss.or.kr/api/list.json
- 전체 재무제표: GET https://opendart.fss.or.kr/api/fnlttSinglAcntAll.json
- 필요하면 공식 주요 재무지표: GET https://opendart.fss.or.kr/api/fnlttSinglIndx.json
- 회사 식별자는 8자리 corp_code, 상장 종목은 6자리 stock_code를 사용한다.
- K-IFRS 연결재무제표 CFS를 우선하고 없으면 별도재무제표 OFS로 폴백한다.
- 사업보고서 코드 11011을 사용하고 최근 연도가 아직 없으면 직전 연도를 재시도한다.
- 공시 원문 URL은 접수번호를 이용해
  https://dart.fss.or.kr/dsaf001/main.do?rcpNo={rcept_no} 형식으로 연결한다.
- API 키가 포함된 요청 URL을 로그에 절대 출력하지 않는다.

### 미국

- SEC EDGAR의 공개 Submissions API와 Company Facts API를 사용한다.
- 회사 식별자는 0을 채운 10자리 CIK를 내부 기준키로 사용한다.
- 제출 이력: https://data.sec.gov/submissions/CIK##########.json
- XBRL 재무: https://data.sec.gov/api/xbrl/companyfacts/CIK##########.json
- SEC_USER_AGENT 환경변수에 서비스명과 실제 연락 이메일을 넣어 모든 요청에 보낸다.
- 전체 서버 합계 초당 10회보다 낮게 제한하고 재시도에는 지수 백오프를 사용한다.
- data.sec.gov는 CORS를 제공하지 않으므로 브라우저가 아닌 서버에서만 호출한다.
- 10-K, 10-Q, 8-K와 수정 보고서를 수집한다.
- XBRL 값은 form, fp, start, end, filed, accession, unit을 검증한다.
- 같은 기간의 값이 여러 번 제출됐으면 가장 최근 제출된 값을 사용하되 수정 이력을
  별도로 식별할 수 있게 한다.

## 4. 일일 갱신 파이프라인

1. 매일 오전 7시 KST에 서버 스케줄러 또는 GitHub Actions 중 하나가 실행된다.
2. 공급자 키가 없으면 해당 국가를 건너뛰고 마지막 정상 스냅샷을 보존한다.
3. DART 기업 코드와 SEC CIK 매핑으로 설정된 추적 대상을 확인한다.
4. 최근 공시와 연간 XBRL 재무를 수집한다.
5. 표준 계정 alias를 사용해 매출, 영업이익, 순이익, 자산, 부채, 자본, 유동자산,
   유동부채, 영업현금흐름, 설비투자를 정규화한다.
6. 단위와 기간을 검증한 뒤 파생 지표를 계산한다.
7. 정규화에 실패한 회사는 기존 정상 데이터를 지우지 말고 stale=true와 오류 사유를
   기록한다.
8. 성공한 데이터만 점수를 다시 계산하고 임시 파일에 쓴 뒤 원자적으로 교체한다.
9. 공급자별 성공·실패 회사 수와 오류 메시지를 메타데이터에 남긴다.
10. 동시 실행을 막는 잠금 파일을 사용한다.

전체 시장으로 확장할 때는 SEC Company Facts bulk ZIP과 변경 CIK 목록을 사용하고,
DART는 다중회사 API와 증분 공시 수집을 사용한다. 모든 회사를 매일 회사별로 전량
재호출하지 않는다.

## 5. 파생 재무지표

- ROE = 최근 연간 순이익 / 최근·전기 평균 자본 × 100
- 영업이익률 = 영업이익 / 매출 × 100
- 순이익률 = 순이익 / 매출 × 100
- 매출 성장률 = (최근 매출 - 전기 매출) / 전기 매출 × 100
- 영업이익 성장률은 전기 영업이익이 양수일 때만 같은 방식으로 계산한다.
- 부채비율 = 부채 / 자본 × 100. 자본이 0 이하이면 누락이 아니라 별도 위험 상태로
  취급하는 확장을 준비한다.
- 유동비율 = 유동자산 / 유동부채 × 100
- FCF = 영업현금흐름 - 설비투자 절댓값
- FCF 마진 = FCF / 매출 × 100
- 현금 전환율 = 영업현금흐름 / 순이익 × 100. 순이익이 0 이하이면 비율을 만들지 않는다.
- 이익이나 현금흐름의 시작값이 음수이면 CAGR을 만들지 않는다.
- 회사가 제시한 비GAAP 조정 이익은 기본 점수에 사용하지 않는다.

## 6. 점수 모델 v1

일반 비금융회사 전용 100점 모델로 구현한다.

- 수익성 25점: ROE 10, 영업이익률 8, 순이익률 7
- 성장성 20점: 매출 성장률 10, 영업이익 성장률 10
- 안정성 20점: 부채비율 12, 유동비율 8
- 현금흐름 20점: FCF 마진 12, 현금 전환율 8
- 지속성 10점: 최근 흑자 연도 수 6, 매출 성장 안정성 4
- 공시신뢰 5점: 최근 공시 최신성 3, 정정 횟수 2

각 지표는 공개된 weak와 strong 절대 구간을 0~100으로 선형 변환하고 범위를 벗어나면
0 또는 100으로 제한한다. 지표가 누락되면 사용 가능한 배점만으로 임시 원점수를
구하되, 최종점수는 다음처럼 중립점으로 수렴시킨다.

~~~text
완전성 = 유효 지표 배점 / 100
최종점수 = 50 + (임시 원점수 - 50) × 완전성
~~~

향후 분석대상이 국가·공통산업군별 최소 20개 이상 확보되면 v2에서 절대 구간과
국가·산업 백분위를 병행한다. 은행·보험·REIT는 v1 일반 순위에서 제외하고 전용 모델을
만들기 전에는 “평가 보류”로 표시한다.

## 7. 데이터 신뢰도와 후보 자격

데이터 신뢰도는 지표 커버리지 50%, 공시 최신성 20%, 회계식·기간·단위 검증 20%,
원문 추적성 10%로 별도 계산한다.

장기 검토 후보는 다음 조건을 모두 만족해야 한다.

- 종합 분석점수 75점 이상
- 데이터 신뢰도 80 이상
- 지표 완전성 80% 이상
- 연차 이력 3개 이상
- fixture가 아닌 live 데이터
- 중대 위험 플래그 없음

중대 플래그 예시는 연차보고서 제출 지연, SEC 8-K Item 4.02, 감사의견 문제,
계속기업 불확실성, 중요한 재작성, 채무불이행, 회생·파산, 상장폐지 결정이다.
초기 버전에서 구조화 필드로 확정할 수 없는 NLP 위험은 감점하지 말고 “원문 확인 필요”
경고로만 표시한다.

## 8. 설명 생성

- 구조화 지표 중 기여도가 큰 서로 다른 영역의 긍정 근거를 최대 두 개 고른다.
- 가장 약한 영역 또는 위험 요인을 최소 한 개 함께 표시한다.
- “확실하다”, “무조건”, “수익 보장”, “강력 매수” 같은 문구를 금지한다.
- LLM을 사용한다면 문장 다듬기만 맡기고 회사명·숫자·기간·출처는 구조화 데이터와
  다시 대조한다.

## 9. UI 방향

- 신뢰감 있는 딥 그린, 따뜻한 오프화이트, 라임 포인트를 사용한다.
- 과도한 트레이딩 화면처럼 만들지 말고 장기 리서치 문서와 스크리너의 중간 느낌으로
  디자인한다.
- 데스크톱 회사 행은 순위, 회사·근거, 원지표와 점수 막대, 연간 재무 막대,
  종합점수와 신뢰도의 5개 영역으로 구성한다.
- 모바일에서는 1열 카드로 재배치한다.
- 키보드 검색 단축키, 명확한 focus 스타일, dialog 접근성, 색상 외 텍스트 상태를
  제공한다.
- 로딩, 빈 결과, API 오류, 데모, 부분 실패, stale 상태를 모두 디자인한다.

## 10. 보안과 운영

- 외부 API 키를 프런트엔드 번들, URL 로그, 저장소에 넣지 않는다.
- 수동 동기화 API는 별도 Bearer 토큰으로 보호하고 토큰이 없으면 비활성화한다.
- 정적 파일 경로 순회를 막고 CSP, nosniff, frame 차단 헤더를 설정한다.
- 공급자 API 장애 시 마지막 정상 데이터를 제공하되 stale 표시를 제거하지 않는다.
- GitHub Actions에는 최소 contents:write 권한만 주고 secrets로 환경변수를 전달한다.
- 유료화, 개인화 포트폴리오, 매수 시점·가격, 양방향 상담을 추가하기 전에 한국
  금융규제 전문가의 검토를 받는다.

## 11. 필수 면책

다음 취지의 문구를 화면 하단과 방법론에 표시한다.

“본 서비스는 공개 공시자료를 정해진 규칙으로 분석한 정보·교육용 연구 도구입니다.
특정 이용자의 재무상황, 투자목적 또는 위험성향을 고려하지 않으며 매수·매도·보유
권유나 미래 수익을 보장하지 않습니다. 공시 데이터에는 지연, 오류 및 사후 정정이
있을 수 있습니다. 투자에는 원금 손실 가능성이 있으므로 결정 전 원문 공시를 확인하고
필요하면 자격 있는 전문가와 상담하십시오.”

## 12. 완료 조건

- 외부 런타임 의존성 없이 Node.js 20 이상에서 실행된다.
- npm start로 사이트를 열고 npm run sync로 공식 데이터를 갱신할 수 있다.
- DART 키만 있어도 한국 데이터가, SEC User-Agent만 있어도 미국 데이터가 갱신된다.
- 한 공급자 실패가 다른 국가와 기존 정상 스냅샷을 삭제하지 않는다.
- 검색·국가·산업·후보 필터와 정렬이 동작한다.
- 점수, 완전성, 신뢰도, 후보 자격, 원문 링크가 화면에서 구분된다.
- PER 미연동 상태가 대시로 표시되고 점수에 들어가지 않는다.
- 점수 경계, 결측, 중대 플래그, 데모 후보 제외를 자동 테스트한다.
- 매일 실행되는 워크플로와 환경변수 설정 문서를 제공한다.

이 프롬프트를 구현할 때 화면의 멋보다 데이터 계보와 실패 시 정직한 상태 표시를
우선하라. 최종 결과는 투자 결론을 대신하는 추천기가 아니라 원문 공시로 되돌아갈 수
있는 투명한 리서치 도구여야 한다.
