# Longview 일일 시세·가치평가 보강 구현 프롬프트 v1

> 이 문서는 시세 기능을 처음 구현한 당시의 기록입니다. 현재 점수 기준은
> `MASTER_PROMPT.md`의 가치·장기성장 모델 v2가 우선합니다.

아래 프롬프트는 기존 공시 기반 Longview에 한국·미국 상장기업의 일일 종가와 검증 가능한
추가 재무지표를 연결하기 위한 실행 명세다. 구현 순서는 이 문서를 먼저 확정한 뒤 코드,
테스트, 자동화, 화면 순으로 진행한다.

---

당신은 금융 데이터 엔지니어, 퀀트 리서처, 백엔드 엔지니어이자 제품 디자이너다.
기존 Longview의 공시 원문 추적성, 결측의 정직한 표시, 실패 시 마지막 정상 데이터 보존
원칙을 유지하면서 다음 요구사항을 모두 구현하라.

## 1. 목표와 사실 기준

- 한국 KOSPI·KOSDAQ 및 미국 Nasdaq·NYSE·Cboe 목록에 최신 확인 가능한 일봉 종가,
  등락률, 거래량, 거래대금 또는 시가총액을 표시한다.
- "실시간"이라고 과장하지 않는다. 한국 금융위원회 주식시세정보는 기준일 다음 영업일
  오후 1시 이후 갱신되는 T+1 공식 종가다. 화면에 공급자와 가격 기준일을 함께 표시한다.
- 가격이 없는 경우 `0`이나 추정값으로 채우지 않고 `N/A`로 둔다. 오래된 가격은
  `stale`, 키가 없으면 `not_configured`, 적법한 원천이 없으면 `unavailable`로 구분한다.
- 주가는 공시 갱신 시각과 별도 수명주기를 가진다. 가격 수집 실패 때문에 회사의 DART/SEC
  공시 상태를 `stale`로 바꾸지 않는다.
- 투자 권유가 아니라 공시와 시세를 정리한 연구 도구임을 유지한다.

## 2. 한국 시세 원천

- 금융위원회 공공데이터포털 `주식시세정보` Open API를 사용한다.
  - 문서: `https://www.data.go.kr/data/15094808/openapi.do`
  - 서비스: `GetStockSecuritiesInfoService/getStockPriceInfo`
- 인증키는 `DATA_GO_KR_API_KEY` 환경변수에서만 읽고 로그, 오류, URL, JSON에 기록하지
  않는다. GitHub에서는 Repository Secret으로만 전달한다.
- 오늘부터 과거로 기준일을 역탐색해 처음으로 결과가 있는 영업일을 선택한다. 한 날짜의
  모든 페이지를 `totalCount`까지 읽고 KOSPI·KOSDAQ만 남긴다.
- DART 회사의 6자리 `stockCode`와 API의 `srtnCd`를 정확히 일치시킨다. 회사명 유사
  매칭은 금지한다.
- 응답의 종가, 시가, 고가, 저가, 공식 등락률, 거래량, 거래대금, 상장주식수, 시가총액을
  유한한 양수·0 규칙에 맞게 검증한다. 중복 코드, 미래 날짜, 비정상 커버리지는 거부한다.
- 정상 커버리지가 기준보다 급감하면 새 결과를 쓰지 않고 이전 정상 `marketData`를
  보존한다.

## 3. 미국 시세의 라이선스 경계

- 무료 미국 전체시장 API의 개인용 요금제는 대개 원자료와 가격 기반 파생값의 공개
  재배포를 금지한다. Yahoo, Nasdaq 웹 화면 등의 비공식 endpoint를 스크래핑하지 않는다.
- 공개 `data/companies.json`에는 다음 중 하나만 쓸 수 있다.
  1. 공급자 manifest가 원자료와 파생값의 공개 재배포 권리를 명시한 사용자가 보유한
     `licensed-snapshot`.
  2. 가격과 무관한 SEC 공식 Company Facts 보강값.
- 라이선스 manifest에는 `usageMode`, `redistributionAllowed`,
  `derivedPublicationAllowed`, `licenseReference`, `licenseId`, `rightsReviewedAt`을
  요구한다. 두 공개 허용값이 모두
  `true`가 아니면 공개 writer가 가격과 가격 기반 지표 기록을 거부한다.
- 무료 개인용 공급자의 데이터는 암호화 여부와 관계없이 GitHub Actions로 전송하거나
  공개 저장소에 커밋하지 않는다. 무료이면서 미국 전체시장 자동수집·재배포를 명시적으로
  허용하는 원천을 확인하지 못한 상태에서는 미국 가격을 `unavailable`로 둔다.
- 미국 가격은 운영자가 계약·라이선스를 직접 확인하고 승인 host와 공개권리 식별자를
  설정한 `licensed-snapshot`만 선택적으로 받는다. `US_LICENSED_PRICE_ALLOWED_HOSTS`에
  없는 host와 private/link-local 주소는 거부하며 권리를 코드가 추정하지 않는다.
- 미국 전 종목은 거래소와 ticker로만 결합한다. 점과 하이픈 별칭은 정확히 하나의
  종목으로 해소될 때만 허용하며 CIK가 같아도 서로 다른 주식 종류의 시가총액을 함부로
  합산하지 않는다.

## 4. 표준 데이터와 계산

- 회사별 공개 가격은 다음 최소 구조를 사용한다.

```json
{
  "marketData": {
    "status": "ok",
    "freshness": "current",
    "usageMode": "public",
    "ticker": "005930",
    "exchange": "KOSPI",
    "currency": "KRW",
    "asOf": "2026-07-17",
    "fetchedAt": "2026-07-18T10:35:00.000Z",
    "price": 70000,
    "previousClose": 69000,
    "changePercent": 1.45,
    "volume": 10000000,
    "turnover": 700000000000,
    "marketCap": 400000000000000,
    "listedShares": 5900000000,
    "source": { "name": "금융위원회 주식시세정보", "url": "..." },
    "valuation": { "per": 12.3, "pbr": 1.4, "psr": 2.1, "fcfYield": 4.2 }
  }
}
```

- PER은 양수인 같은 기간 순이익이 있을 때 `시가총액 / 순이익`, PBR은 양수인 최신
  자본이 있을 때 `시가총액 / 자본`, PSR은 양수 매출이 있을 때 `시가총액 / 매출`로
  계산한다. 단위와 연결/별도 기준을 확인할 수 없으면 계산하지 않는다.
- FCF 수익률은 `검증된 FCF / 시가총액`으로만 계산한다. FCF는
  `영업현금흐름 - 자본적지출`이며, 가격만으로 FCF 마진이나 현금전환율을 만들지 않는다.
- 음수 순이익의 PER, 음수 자본의 PBR, 0 분모, `NaN`, 무한대는 `null`이다.
- 보강값은 `metricLineage`에 공급자, 기준일, 사용한 공시 기간, 계산식을 남긴다.

## 5. 공시 기반 결측 보강과 점수

- 점수 모델 v1의 수익성 25, 성장성 20, 안정성 20, 현금흐름 20, 지속성 10,
  공시신뢰 5를 유지한다. 가격 도입만으로 총점 가중치를 조용히 바꾸지 않는다.
- DART 전체 재무제표와 SEC Company Facts에서 검증 가능한 영업현금흐름·CAPEX를
  추가 추출해 FCF 마진과 현금전환율을 채운다. 같은 기간·같은 연결 기준이 아니면
  결측으로 둔다.
- 기존 공식 지표를 fallback이 덮어쓰지 않는다. 결측값만 `??=` 원칙으로 보강한다.
- 가격에서 계산한 PER·PBR·PSR·FCF 수익률은 별도의 가치평가 참고 영역에 표시한다.
  이 버전에서는 총점에 넣지 않고 방법론 API에 그 사실을 공개한다.
- 누락 지표의 배점을 다른 지표에 몰아주지 않는 기존 완전성 공식을 유지한다.

## 6. 화면

- 목록 카드에는 통화 단위가 붙은 최신 종가, 등락률, 가격 기준일과 PER을 간결하게
  보여 준다. 가격이 없으면 이유가 있는 `N/A`를 표시한다.
- 상세 화면에는 종가, 시가총액, PER, PBR, PSR, FCF 수익률과 공급자·기준일·상태를
  표시한다. 공시 기준일과 시세 기준일을 혼동하지 않게 분리한다.
- 총점의 `100 /100`이 겹치지 않도록 suffix를 별도 inline block으로 만들고 음수
  letter-spacing을 상속하지 않게 한다. 100점, 두 자릿수, 모바일 폭을 확인한다.
- 보존 가격과 10일을 넘긴 가격은 각각 `마지막 정상 가격`, `오래된 가격`으로 표시한다.

## 7. 일일 자동화와 실패 안전성

- 기존 GitHub Actions의 DART/SEC 병합 뒤 시세 보강 단계를 실행한다.
- 한국 키가 없거나 공급자가 실패하면 전체 공시 동기화를 실패시키지 않고 이전 한국
  시세를 보존한다. 실패 요약은 secret이 제거된 진단 파일과 meta에만 남긴다.
- 공개 라이선스 snapshot은 schema, 날짜, 고유 ticker, 커버리지, 라이선스 manifest를
  통과할 때만 병합한다.
- 새 가격 날짜가 기존보다 오래되면 덮어쓰지 않는다. 부분 실패한 국가의 마지막 정상
  시세를 보존한다.
- 자동 커밋 대상은 재배포가 허용된 공개 `data/companies.json`으로 명시적으로 제한한다.
- 비밀값, 개인용 미국 원자료, 요청 URL 전체, 응답 원문은 artifact나 로그에
  남기지 않는다.

## 8. 완료 조건

- API 응답 정규화, 앞자리 0 한국 코드, 미국 점/하이픈 ticker, 음수·0·NaN 거부,
  중복, 오래된 응답, 낮은 커버리지, 실패 시 보존을 단위 테스트한다.
- PER/PBR/PSR/FCF 수익률의 양수 분모 규칙과 공식 지표 우선 규칙을 테스트한다.
- 공개 writer의 라이선스 거부, 승인되지 않은 host, 오래되거나 혼합된 기준일을 테스트한다.
- 목록 API는 가벼운 가격 필드만 내보내고 상세 API는 출처와 계보를 포함한다.
- 기존 점수 회귀 테스트, `npm run check`, Python 테스트, 스냅샷 validator를 통과한다.
- GitHub 기능 브랜치에 의도한 파일만 커밋하고 푸시한 뒤 수동 Actions 실행으로 실제
  워크플로가 성공하는지 확인한다. 공급자 키가 없는 단계는 실패가 아니라 검증 가능한
  `not_configured` 결과여야 한다.
- README에는 무료로 가능한 범위, 한국 T+1 지연, 미국 공개 재배포 제한, 사용자가 한 번
  발급해야 하는 키와 GitHub Secret 이름을 쉬운 한국어로 적는다.
