# Longview 국내 일일 시세·가치평가 프롬프트

기존 DART 공시 스냅샷에 금융위원회 공식 일일 시세를 연결하고 검증 가능한 가치지표를
계산하라.

## 시세 원천과 수집

- 공공데이터포털 `주식시세정보` Open API를 사용한다.
- 인증키는 `DATA_GO_KR_API_KEY`에서만 읽고 로그·진단·공개 JSON에 남기지 않는다.
- 오늘부터 과거로 기준일을 탐색해 결과가 있는 최근 영업일의 모든 페이지를 수집한다.
- KOSPI·KOSDAQ만 남기고 DART의 6자리 종목코드와 정확히 일치시킨다.
- 종가, 시가, 고가, 저가, 등락률, 거래량, 거래대금, 상장주식수, 시가총액을 유한한
  숫자 규칙으로 검증한다.
- 한 기준일 안의 중복 종목코드, 미래 날짜, 비정상 커버리지는 거부한다.

## 공개 구조와 계산

회사별 `marketData`에는 상태, freshness, 종목코드, 거래소, `KRW`, 기준일, 수집시각,
가격·거래량·시가총액, 공급자와 가치지표를 기록한다.

- PER = 양수 시가총액 / 같은 기준의 양수 순이익
- PBR = 양수 시가총액 / 같은 기준의 양수 자본
- PSR = 양수 시가총액 / 같은 기준의 양수 매출
- FCF 수익률 = 같은 기간의 검증된 FCF / 시가총액

FCF는 같은 연결 기준의 영업현금흐름에서 설비투자를 뺀 값만 사용한다. 음수 순이익의
PER, 음수 자본의 PBR, 0 분모, `NaN`, 무한대는 `null`이다. 계산에는 공급자, 가격 기준일,
공시 기간과 식을 계보로 남긴다.

## 점수와 화면

- 검증된 PER·PBR·PSR·FCF 수익률만 저평가 영역에 반영한다.
- 값이 없으면 0점으로 만들거나 다른 지표의 배점을 늘리지 않는다.
- 카드에는 원화 종가, 등락률, 기준일과 PER을 표시한다.
- 상세 화면에는 종가, 시가총액, 가치지표, 공급자·기준일·상태를 표시한다.
- 보존된 가격과 10일을 넘긴 가격을 각각 명확한 상태로 구분한다.
- 종합점수의 `/100` suffix는 점수 숫자와 겹치지 않게 별도 inline 요소로 배치한다.

## 실패 안전성과 완료 조건

- 시세 키가 없거나 API가 실패해도 DART 공시 갱신을 잃지 않는다.
- 새 가격이 과거보다 오래됐거나 커버리지가 급감하면 마지막 정상 가격을 보존한다.
- 비밀값이 제거된 요약 진단만 저장한다.
- 앞자리 0 종목코드, 음수·0·비정상 숫자, 중복, 오래된 응답, 공급자 실패 시 보존,
  가치지표 분모 규칙을 자동 테스트한다.
- `npm run check`, 가격 검증, 스냅샷 validator를 모두 통과해야 한다.

실시간 호가라고 표현하지 말고 공식 일일 종가의 기준일과 T+1 제공 특성을 안내하라.
