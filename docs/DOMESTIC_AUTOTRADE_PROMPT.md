# 국내 가치·장기 자동투자 구현 프롬프트

아래 요구사항을 하나의 작업으로 구현한다. 기존 Longview 국내 공시·기업 순위 사이트와
현재 모의투자 원장은 보존하고, 자동투자 서비스만 안전하게 확장한다. 구현 뒤에는 코드,
테스트, 문서와 실제 모의 실행 결과를 함께 검증한다.

## 목표

- 투자 대상은 한국거래소의 KOSPI·KOSDAQ 보통주와 원화 현금뿐이다.
- 매일 최신 국내 공시·재무·가격 스냅샷을 평가하되, **가치가 낮게 평가되어 있고 장기
  성장성과 재무 건전성이 뒷받침되는 기업**을 우선한다.
- 투자계좌에 넣은 돈은 운용자금이므로 목표 현금비중을 `0%`로 둔다. 고정 현금 예비금이나
  국가별 예비자금은 두지 않는다.
- 다만 1주 단위, 예상 수수료·세금, 최소주문금액, 가격 제한과 안전조건 때문에 살 수 없는
  금액은 불가피한 잔액으로 인정한다. 신용·미수·대출, 소수점 주문, 안전기준 완화로 현금을
  억지로 소진하지 않는다.
- 한국투자증권(KIS) 공식 Open API의 국내주식 기능으로 모의계좌와 실전계좌를 지원하되,
  기본값에서는 절대로 실전 주문이 나가지 않게 한다.
- 은행 송금·출금은 자동화하지 않는다.

## 변경 금지 및 호환성

- 기존 `:4173` 국내 순위 사이트, DART 수집·점수 계산, GitHub 일일 데이터 갱신을 손상시키지
  않는다. 자동투자는 별도 서비스와 별도 상태 저장소를 계속 사용한다.
- 미국 종목, USD, SEC, 미국 거래소·시세·주문 분기를 새로 만들거나 되살리지 않는다.
- `.autotrade/`의 현재 모의 현금·보유종목·주문·감사 기록을 초기화하거나 임의로 다시 쓰지
  않는다. 필요한 스키마 변경은 하위 호환 방식으로 처리한다.
- `.env`, API 키, 계좌번호와 토큰을 출력·커밋·응답 본문에 노출하지 않는다.

## 종목 선정

1. 기존 가치·장기 점수를 단일 진실 원천으로 사용한다. 평가 가능한 국내 기업만 받고,
   신뢰도·완전성·가치평가 신뢰도·유동성·재무 안전성의 기존 하드 게이트를 유지한다.
2. 단순히 상위 몇 종목을 무조건 채우지 않는다. 적격 후보 중 점수, 업종 분산, 가격과 계좌
   규모를 함께 고려해 설정된 최소·최대 종목 수 안에서 포트폴리오를 만든다.
3. 기본 목표비중은 동일비중으로 두고 점수 차이는 작은 기울기만 반영한다. 한 종목과 한
   업종의 최대비중을 강제하고, 캡 적용 후 남은 비중을 다른 적격 종목에 반복 배분해 목표
   비중의 합이 100%가 되게 한다.
4. `선택 종목 수 × 종목당 최대비중 < 100%`처럼 전액 배분이 수학적으로 불가능한 설정은
   시작 시 명확한 오류로 거부한다. 기본 종목 수와 최대비중도 서로 양립하도록 조정한다.
5. 적격 종목이 너무 적으면 기준을 낮추지 않는다. 그 경우에만 안전상 현금을 남기고 원인을
   계획과 대시보드에 표시한다.

## 0% 목표 현금 배분기

- 주문 가능 현금은 원화 예수금에서 미체결 매수 예약금과 예상 비용을 제외해 계산한다.
- 목표 금액을 1주 단위 수량으로 내림 배정한 뒤, 남은 현금으로 다음 1주와 비용을 감당할 수
  있는 동안 가장 목표비중보다 부족한 적격 종목에 1주씩 추가하는 결정론적 잔액 소진
  알고리즘을 사용한다.
- 추가 배정도 종목·업종 한도, 주문금액 한도, 가격 신선도와 안전조건을 모두 지켜야 한다.
- 결과 현금은 `0원`을 강제하는 값이 아니라, 모든 안전조건을 만족하는 **다음 최소 주문을
  더 이상 만들 수 없는 금액**이어야 한다. 계획에는 목표 현금 0%, 예상 잔액, 잔액이 남는
  구체적 이유를 표시한다.
- 최초 또는 현금 과다 계좌는 검증된 계획에 한해 한 번의 초기 배치 주기에서 가용자금의
  최대 100%까지 투자할 수 있다. 이 예외는 일반 리밸런싱 회전율 한도를 무력화하지 않는다.
- 매도대금은 체결과 잔고를 다시 확인한 뒤 같은 주기의 적격 매수에 재배분한다. 미확인
  매도대금을 미리 쓸 수 있는 돈으로 가정하지 않는다.

## 매일 평가하되 낮은 회전율 유지

- KST 기준 하루 한 번, 새 스냅샷 식별자가 있을 때만 계획을 계산한다. 같은 날짜·스냅샷·
  계좌 조합은 중복 실행하지 않는다.
- 보유종목이 계속 적격이고 목표비중 허용범위 안이면 아무 주문도 만들지 않는다.
- 신규 후보는 기존 보유종목보다 설정된 점수 차이 이상 좋아야 교체할 수 있다. 일반적인
  품질 탈락은 서로 다른 최신 스냅샷에서 2회 연속 확인하고, 단순 수집 실패나 결측만으로
  자동매도하지 않는다.
- 목표비중 이탈은 절대 편차와 상대 편차 기준을 모두 적용하고, 최소주문금액보다 작은 조정은
  누적 관찰만 한다. 일반 주기의 일일·월간 회전율과 주문금액 한도를 둔다.
- 상장폐지·거래정지·계좌 안전 위반 같은 명시적 예외를 제외하면 고정 수익률 도달만으로
  전량 매도하지 않는다. 과대비중은 분할 축소하고 가치·성장 논리가 훼손될 때 교체한다.
- 데이터가 오래됐거나 검증에 실패하면 새 매수와 자동매도를 모두 멈추고 마지막 정상
  포트폴리오를 유지한다.

## KIS 모의·실전 준비

- KIS 공식 국내주식 API만 사용한다. 잔고, 주문가능금액, 현재가, 국내 지정가 매수·매도,
  주문 조회와 체결 확인을 각각 명확히 분리한다.
- `disabled → 로컬 paper → KIS 모의(vps) → KIS 실전(prod)` 순서를 유지한다. API 키만
  입력했다고 실전이 켜지면 안 된다.
- 실전은 다음 잠금이 모두 일치할 때만 가능하다: 실전 모드, KIS 실전 환경, 실전 허용 플래그,
  정확한 위험 확인문구, 0보다 큰 자본 한도, 전용계좌 확인. 무인 실전은 별도 허용 플래그와
  별도 확인문구가 추가로 필요하다.
- 실전 전용계좌에 프로그램이 관리하지 않는 종목, 알 수 없는 미체결 주문 또는 잔고 불일치가
  있으면 전체 주문을 차단한다. 자본 한도를 넘는 예수금은 배분 대상에서 제외한다.
- 정규장·영업일·시세 신선도·호가 제한을 확인하고 지정가만 사용한다. 매도 주문을 확인·정산한
  뒤 매수를 계획하며, 주문 타임아웃이나 결과 `unknown`은 자동 재시도하지 않고 전체 주기를
  중단한다.
- 프로세스 단일 실행 잠금, 주문 직전 및 각 주문 사이의 긴급정지 확인, 종목·업종·주문·일일
  회전율 한도, 최대 허용 가격 괴리를 적용한다.
- 실전 수동 실행은 매번 터미널 확인문구가 필요하고 대시보드에서는 실행할 수 없게 한다.
  무인 실전은 모의계좌 장기 검증 뒤 명시적으로 잠금을 풀 때만 동작한다.

## 감사·복구 가능성

- 계획과 실행마다 스냅샷 ID·평가일·회사 점수·선정/제외 이유·현재/목표 비중·예상 비용·
  주문 전후 잔고·KIS 주문번호·체결상태·차단 사유를 비밀값 없이 기록한다.
- 실행키는 계좌 식별 해시, 날짜, 스냅샷과 계획 내용으로 결정론적으로 만들고 이미 처리한
  실행키는 다시 주문하지 않는다.
- 상태 파일은 원자적으로 저장하고 손상 시 마지막 정상본으로 진단 가능하게 한다. 실패한
  주문을 성공으로 기록하지 않으며, 미확인 주문은 사용자가 증권사 앱과 대조해 명시적으로
  해소하기 전까지 다음 주문을 막는다.
- 로그와 API 응답에서 App Key, App Secret, 토큰, 계좌번호, 개인 식별값을 마스킹한다.

## 필수 테스트

- 배분기: 목표 현금 0%, 목표비중 합 100%, 정수 주식 수량, 예상 비용, 잔액 소진, 종목·업종
  한도, 설정 불가능 감지, 적격 후보 부족, 비싼 주식과 최소주문 경계, 결정론을 검증한다.
- 전략: 동일 스냅샷 무주문, 우량 보유 유지, 점수 차이·2회 확인·비중 이탈·최소주문·회전율,
  최초 100% 배치와 매도대금 재투자를 검증한다.
- 안전: 데이터 지연·결측 시 거래 중지, 중복 실행 차단, 긴급정지, 장 마감, 미체결·알 수 없는
  주문, 예상과 다른 잔고, 자본 한도 초과를 검증한다.
- KIS: 모든 네트워크 호출을 모킹해 모의/실전 URL과 거래 ID, 국내 잔고 페이지네이션,
  지정가 주문, 체결 재조회, 타임아웃 무재시도와 비밀값 마스킹을 검증한다.
- 통합: 100만원 로컬 모의계좌에서 자금 보존식이 맞고, 최종 현금이 안전한 다음 최소 주문보다
  작은지 검증한다. 같은 입력 재실행 시 중복 체결이 없어야 하며 기존 모의 원장은 보존한다.
- 회귀: 국내 순위 사이트·DART·국내 시세 테스트를 모두 통과하고 코드·UI·API·문서·데이터에
  미국 투자 경로가 없어야 한다.

## 완료 기준

1. `npm.cmd run check`와 자동투자 전체 테스트가 통과한다.
2. 계획 화면에 목표 현금 `0%`, 100%로 정규화된 목표비중, 예상 투자금과 불가피한 잔액 이유가
   표시된다.
3. 기존 모의 원장을 이어서 한 번 실행했을 때 기존 종목을 불필요하게 매도하지 않고, 적격
   종목·한도 안에서 현금을 최대한 투자한다. 남은 현금으로 가능한 안전한 주문이 없어야 한다.
4. 다음 날 점수가 같으면 거래가 없고, 더 좋은 후보나 확정된 악화가 있을 때만 한도 내에서
   교체·재배분한다.
5. KIS 모의계좌는 계획·주문·체결·잔고가 대조되고, 실전 잠금을 하나라도 빼면 주문이 반드시
   차단된다.
6. 실제 주문 전환 방법, 중단·복구 방법, 잔액이 남을 수 있는 이유와 손실 가능성을 README와
   자동투자 안내 문서에 반영한다.

구현 완료 보고에는 변경 파일, 실제로 실행한 테스트, 모의 실행 전후 현금·보유비중, 남은
잔액의 이유, 아직 사용자가 직접 해야 하는 KIS 키·전용계좌 설정만 간단히 적는다. 수익을
보장한다고 표현하지 않는다.
